# Contributing to OpenMates

Thank you for wanting to help improve OpenMates.

OpenMates is currently a single-person project, so the most helpful contributions right now are clear bug reports, feature requests, improvement suggestions, and discussion around what should be built next.

Code contributions are welcome in principle, but I am not actively optimizing for external pull requests yet. I still need to improve the development setup, contributor guidelines, and review process before encouraging larger code contributions.

## What Helps Most Right Now

The best ways to contribute right now are:

1. Report bugs you find while using OpenMates
2. Suggest features or improvements
3. Join the Signal group for development discussions
4. Test the cloud or self-hosted version and share feedback
5. Improve documentation or report unclear documentation
6. Share OpenMates with people who may be interested

## Report A Bug

Bug reports are the most useful contribution right now.

Please include:

- What you were trying to do
- What happened
- What you expected to happen
- Whether the issue happens every time or only sometimes
- Your browser, device, and operating system if relevant
- Screenshots or screen recordings if they help explain the issue

### If You Use OpenMates Cloud

Please use the built-in issue reporting functionality inside OpenMates.

Press the report issue button that looks like a bug in the top-left corner of the app. That is the preferred way to report cloud issues because it can include useful app context that makes debugging easier.

### If You Use The Self-Hosted Edition

Please create a GitHub issue:

https://github.com/glowingkitty/OpenMates/issues

For self-hosted bugs, please include:

- The OpenMates version or commit you are using, if known
- Your deployment setup
- Relevant logs or error messages
- Steps to reproduce the issue
- Any local changes you made

## Suggest A Feature Or Improvement

Feature requests and improvement ideas are very welcome.

Please create a GitHub issue:

https://github.com/glowingkitty/OpenMates/issues

Helpful feature requests usually explain:

- The problem you want solved
- Who the feature would help
- How you imagine it working
- Any alternatives or workarounds you already tried
- Whether this is important for cloud users, self-hosted users, or both

Small improvement suggestions are also useful. If something feels confusing, slow, unclear, or unnecessarily complicated, please open an issue.

## Code Contributions

I am not actively focusing on external code contributions yet.

Pull requests are still welcome for small, clearly scoped changes, especially:

- Documentation fixes
- Typo fixes
- Small bug fixes linked to an existing issue
- Simple improvements that do not require major product decisions

Before working on larger code changes, please open a GitHub issue or discuss the idea in the Signal group first. This avoids duplicated work and makes sure the change fits the current direction of the project.

More complete development guidelines are coming later.

Existing technical docs are available here:

- Contributor docs: `docs/contributing/README.md`
- Frontend standards: `docs/contributing/standards/frontend.md`
- Backend standards: `docs/contributing/standards/backend.md`
- Testing guide: `docs/contributing/guides/testing.md`

## Documentation Contributions

Documentation improvements are very helpful.

Good documentation contributions include:

- Fixing outdated information
- Making setup instructions clearer
- Improving self-hosting docs
- Explaining confusing concepts
- Adding missing troubleshooting steps

For small fixes, a pull request is fine. For larger documentation changes, please open an issue first.

https://github.com/glowingkitty/OpenMates/issues
