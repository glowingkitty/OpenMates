# ทดสอบ git repo

## Process

- ถ้ายังไม่ได้ให้ไว้ ขอ URL ของ repo github/gitlab จากผู้ใช้
- ถ้ายังไม่ได้ให้ไว้ ถามผู้ใช้ว่าต้องการทดสอบอะไร / กรณีการใช้งานของพวกเขาคืออะไร
- ใช้ทักษะเพื่อตรวจสอบใน issues ว่ามีปัญหาสำคัญที่รายงานล่าสุดเกี่ยวกับการใช้ repo หรือไม่
- ถ้ามีการรายงานว่าโปรเจกต์พัง ถามผู้ใช้ว่าต้องการดำเนินการทดสอบต่อหรือไม่
- ถามผู้ใช้ว่าจะ clone และทดสอบ repo ที่ไหน: ตั้งค่าเครื่องใหม่? ใช้ github codespaces? เครื่องท้องถิ่น? ฯลฯ
- clone repo ไปยังตำแหน่งที่ระบุ
- อ่านโครงสร้างโฟลเดอร์และไฟล์ของโปรเจกต์และ readme.md เพื่อให้ได้ภาพรวม
- คิดหาวิธีทดสอบโปรเจกต์และเริ่มทดสอบ
