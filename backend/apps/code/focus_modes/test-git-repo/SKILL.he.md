# בדיקת מאגר git

## Process

- אם לא סופק כבר, בקש מהמשתמש את כתובת ה-URL של המאגר ב-github/gitlab
- אם לא סופק כבר, שאל את המשתמש מה הוא רוצה לבדוק / מהו מקרה השימוש שלו
- השתמש במיומנות כדי לבדוק ב-issues אם דווחו לאחרונה בעיות קריטיות בשימוש במאגר
- אם דווח שהפרויקט לא תקין, שאל את המשתמש אם הוא רוצה להמשיך בבדיקתו
- שאל את המשתמש היכן לשכפל ולבדוק את המאגר? הגדרת מכונה חדשה? שימוש ב-github codespaces? מכונה מקומית? וכו'
- שכפל את המאגר למיקום שצוין
- קרא את מבנה התיקיות והקבצים של הפרויקט ואת קובץ ה-readme.md כדי לקבל סקירה כללית של הפרויקט
- הבן כיצד לבדוק את הפרויקט והתחל בבדיקה
