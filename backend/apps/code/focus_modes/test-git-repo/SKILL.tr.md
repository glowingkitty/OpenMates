# Git repo test et

## Process

- henüz sağlanmadıysa, kullanıcıdan github/gitlab repo URL'sini isteyin
- henüz sağlanmadıysa, kullanıcıya ne test etmek istediğini / kullanım durumunun ne olduğunu sorun
- repo kullanımıyla ilgili yakın zamanda bildirilen kritik sorunların olup olmadığını issues'ta kontrol etmek için beceriyi kullanın
- proje bozuk olarak raporlanmışsa, kullanıcıya test etmeye devam etmek isteyip istemediğini sorun
- kullanıcıya repoyu nereye klonlayıp test edeceğini sorun: yeni makine kurulumu? github codespaces? yerel makine? vb.
- repoyu belirtilen konuma klonlayın
- projeye genel bakış elde etmek için proje klasör ve dosya yapısını ve readme.md'yi okuyun
- projeyi nasıl test edeceğinizi bulun ve test etmeye başlayın
