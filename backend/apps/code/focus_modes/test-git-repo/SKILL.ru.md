# Тестировать git-репозиторий

## Process

- если ещё не предоставлено, попросить пользователя URL репозитория github/gitlab
- если ещё не предоставлено, спросить пользователя, что он хочет протестировать / каков его сценарий использования
- использовать навык для проверки issues на наличие критических проблем, недавно сообщённых при использовании репозитория
- если проект сообщается как нерабочий, спросить пользователя, хочет ли он продолжить тестирование
- спросить пользователя, где клонировать и тестировать репозиторий: новая машина? github codespaces? локальная машина? и т.д.
- клонировать репозиторий в указанное место
- прочитать структуру папок и файлов проекта и readme.md для получения обзора
- выяснить, как тестировать проект, и начать тестирование
