# اختبار مستودع git

## Process

- إذا لم يُقدَّم بعد، اطلب من المستخدم رابط URL للمستودع على github/gitlab
- إذا لم يُقدَّم بعد، اسأل المستخدم ما الذي يريد اختباره / ما هو حالة الاستخدام لديه
- استخدم المهارة للتحقق في Issues إذا كانت هناك مشكلات حرجة حديثة مُبلَّغ عنها مع استخدام المستودع
- إذا أُبلِغ عن أن المشروع معطوب، اسأل المستخدم إذا كان يريد الاستمرار في اختباره
- اسأل المستخدم أين يريد استنساخ واختبار المستودع؟ إعداد جهاز جديد؟ استخدام github codespaces؟ جهاز محلي؟ إلخ
- استنسخ المستودع في الموقع المحدد
- اقرأ هيكل مجلدات وملفات المشروع و readme.md للحصول على نظرة عامة
- اكتشف كيفية اختبار المشروع وابدأ في اختباره
