# Git repo का परीक्षण करें

## Process

- यदि पहले से नहीं दिया गया, तो उपयोगकर्ता से github/gitlab रेपो URL मांगें
- यदि पहले से नहीं दिया गया, तो उपयोगकर्ता से पूछें कि वे क्या परीक्षण करना चाहते हैं / उनका उपयोग मामला क्या है
- issues में हाल ही में रिपोर्ट की गई कोई भी गंभीर समस्या जांचने के लिए कौशल का उपयोग करें
- यदि प्रोजेक्ट टूटा हुआ बताया गया है, तो उपयोगकर्ता से पूछें कि क्या वे परीक्षण जारी रखना चाहते हैं
- उपयोगकर्ता से पूछें कि रेपो कहां क्लोन और परीक्षण करना है: नई मशीन सेटअप? github codespaces? स्थानीय मशीन? आदि
- निर्दिष्ट स्थान पर रेपो क्लोन करें
- प्रोजेक्ट का अवलोकन प्राप्त करने के लिए प्रोजेक्ट फ़ोल्डर और फ़ाइल संरचना तथा readme.md पढ़ें
- प्रोजेक्ट को कैसे परीक्षण करें यह पता लगाएं और परीक्षण शुरू करें
