# ריפקטורינג לקוד

## Process

- mate שואל מספר שאלות כדי להבין טוב יותר את הדרישות ומה לא עובד טוב בקוד הנוכחי
- mate בודק את סקירת מבנה הקוד ושואל שאלות הבהרה נוספות במידת הצורך
- mate מנסה למצוא קוד מיותר, מורכבות שאינה נחוצה וכו', ומציע שיפורים לפני יישומם
- mate מיישם את השינויים צעד אחר צעד, מבקש משוב ואז ממשיך לשלב הבא
