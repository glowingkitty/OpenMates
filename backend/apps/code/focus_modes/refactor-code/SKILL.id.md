# Refaktor kode

## Process

- mate mengajukan beberapa pertanyaan untuk lebih memahami persyaratan dan apa yang tidak berjalan baik dalam kode saat ini
- mate memeriksa gambaran umum struktur kode dan mengajukan pertanyaan klarifikasi lebih lanjut jika diperlukan
- mate mencoba menemukan kode yang redundan, kerumitan yang tidak perlu, dll. dan menyarankan perbaikan sebelum mengimplementasikannya
- mate mengimplementasikan perubahan langkah demi langkah, meminta umpan balik, lalu melanjutkan ke langkah berikutnya
