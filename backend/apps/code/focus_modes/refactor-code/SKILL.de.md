# Code refaktorisieren

## Process

- mate stellt mehrere Fragen, um die Anforderungen und was im aktuellen Code nicht gut funktioniert besser zu verstehen
- mate überprüft die Codestrukturübersicht und stellt bei Bedarf weitere Klärungsfragen
- mate versucht redundanten Code, unnötige Komplexität usw. zu finden und schlägt Verbesserungen vor, bevor sie implementiert werden
- mate implementiert die Änderungen schrittweise, bittet um Feedback und fährt dann mit dem nächsten Schritt fort
