    CHANGE 1 — OMToggle ON state color (CRITICAL):
    The web Toggle.svelte uses `background: var(--color-primary)` which is a GRADIENT (`linear-gradient(135deg, #4867cd, #5a85eb)`).
    The current Swift uses `Color.buttonPrimary` which is ORANGE (#FF553B).
    This means every toggle in every settings page shows an orange track instead of a blue gradient track.

    Fix: Change the track fill from:
    ```swift
    .fill(isOn ? Color.buttonPrimary : Color.grey30)
    ```
    to:
    ```swift
    .fill(isOn ? AnyShapeStyle(LinearGradient.primary) : AnyShapeStyle(Color.grey30))
    ```
    Note: Since `RoundedRectangle.fill()` requires a single ShapeStyle, and we need to switch between a gradient and a solid color, use `AnyShapeStyle` or use a conditional overlay approach:
    ```swift
    RoundedRectangle(cornerRadius: trackHeight / 2)
        .fill(Color.grey30) // OFF state base
        .overlay(
            Group {
                if isOn {
                    RoundedRectangle(cornerRadius: trackHeight / 2)
                        .fill(LinearGradient.primary)
                }
            }
        )
        .frame(width: trackWidth, height: trackHeight)
    ```

    CHANGE 2 — OMToggle track shadow to INSET:
    Web CSS: `box-shadow: inset 0 2px 4px rgba(0, 0, 0, 0.2)` — this is an inset shadow (inner shadow).
    Current Swift: `.shadow(color: .black.opacity(0.2), radius: 2, x: 0, y: 2)` — this is an outer shadow.
    SwiftUI does not have native inset shadow. Replace the outer shadow with an `.innerShadow` overlay:
    ```swift
    .overlay(
        RoundedRectangle(cornerRadius: trackHeight / 2)
            .stroke(Color.black.opacity(0.15), lineWidth: 1)
            .blur(radius: 2)
            .offset(y: 1)
            .mask(RoundedRectangle(cornerRadius: trackHeight / 2))
    )
    ```
    Remove the existing outer `.shadow()` on the track (keep the thumb shadow — thumb's outer shadow is correct per web CSS).

    CHANGE 3 — OMDropdown border radius:
    Web CSS uses `border-radius: 1.5rem` = 24px. Current Swift uses `.radius7` = 16pt.
    There is no 24pt radius token. The closest is `.radius8` = 20pt, but 24px is the correct web value.
    Since the web value (24px -> 24pt) doesn't map to an existing token, use `.radius12` — wait, that's a spacing token (24pt), not a radius token. Use a hardcoded `24` with a comment explaining why:
    ```swift
    .clipShape(RoundedRectangle(cornerRadius: 24)) // 1.5rem from SettingsDropdown.svelte — no matching radius token
    ```
    Apply this to both the closed-state row AND the expanded option list.

    CHANGE 4 — OMDropdown padding:
    Web CSS: `padding: 1.0625rem 3rem 1.0625rem 1.4375rem` = 17pt top/bottom, 48pt right (for chevron space), 23pt left.
    Current Swift uses `.padding(.horizontal, .spacing6)` (12pt) and `.padding(.vertical, .spacing5)` (10pt).
    Update to match web padding more closely:
    ```swift
    .padding(.leading, 23)  // 1.4375rem from SettingsDropdown.svelte
    .padding(.trailing, 48) // 3rem from SettingsDropdown.svelte (chevron space)
    .padding(.vertical, 17) // 1.0625rem from SettingsDropdown.svelte
    ```
    Or use the closest token values: `.spacing8` (16pt) for vertical, custom for horizontal.
    Add comments referencing the web source CSS values.

    CHANGE 5 — OMSettingsRow padding:
    Web CSS: `padding: 0.75rem 0.625rem` = 12pt vertical, 10pt horizontal.
    Current Swift uses `.padding(.horizontal, .spacing6)` (12pt) and `.padding(.vertical, .spacing5)` (10pt).
    The vertical padding is correct (10pt matches 0.625rem); wait, let me re-check:
    - CSS: `padding: 0.75rem 0.625rem` means `padding-top: 0.75rem (12px), padding-right: 0.625rem (10px), padding-bottom: 0.75rem (12px), padding-left: 0.625rem (10px)`
    - So vertical = 12pt, horizontal = 10pt.
    - Current Swift: `.padding(.horizontal, .spacing6)` = 12pt, `.padding(.vertical, .spacing5)` = 10pt
    - These are SWAPPED. Fix: `.padding(.horizontal, .spacing5)` (10pt) and `.padding(.vertical, .spacing6)` (12pt)

    Apply the same fix to OMSettingsToggleRow, OMSettingsPickerRow, OMSettingsStaticRow, and OMSettingsRow — they all use the same padding pattern.

    CHANGE 6 — OMSettingsRow gap:
    Web CSS: `gap: 0.75rem` = 12px = `.spacing6`.
    Current Swift: `HStack(spacing: .spacing4)` = 8pt.
    Fix: Change to `HStack(spacing: .spacing6)` in OMSettingsRow and all row variants.

    CHANGE 7 — OMSettingsRow border-radius:
    Web CSS: `border-radius: 0.5rem` = 8px = `.radius3`.
    This is on the individual row, not the section. Currently the row itself has no explicit corner radius (the section applies `.radius8`=20pt). The row's own radius should be `.radius3` for hover state background. Since SwiftUI button rows don't have independent background rounding currently, this is acceptable as-is (the section wraps them).
