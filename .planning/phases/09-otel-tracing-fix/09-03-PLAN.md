    ```python
    def _get_full_trace_spans(
        trace_id: str,
        start_time_us: int,
        end_time_us: int,
        base_url: str,
        auth: Tuple[str, str],
    ) -> List[Dict[str, Any]]:
        """Fetch ALL spans for a trace via SQL search API."""
        sql = (
            f"SELECT * FROM {TRACE_STREAM} "
            f"WHERE trace_id = '{trace_id}' "
            f"ORDER BY start_time ASC"
        )
        return _search_traces_legacy(sql, start_time_us, end_time_us, base_url, auth)
    ```

    **Part 2: Rework format_trace_timeline**

    Update the formatter to produce the tree output per D-03:

    ```
      ├─ redis.get cache:chats (2ms) OK
      └─ httpx.post app-ai (180ms) OK
          └─ celery.task ask_skill (170ms) OK
    ```

    Changes to format_trace_timeline:
    1. Trace header: Include root span's operation_name (HTTP path or WS message type) instead of just "Duration: Xms"

    3. Use Unicode box-drawing: `\u251c\u2500` for middle children, `\u2514\u2500` for last child
    4. Indent nested children with `\u2502  ` (vertical bar + spaces) for middle siblings, `   ` (spaces) for last sibling

    The recursive _render_span function needs siblings context to choose the right prefix character. Refactor to pass `is_last: bool` parameter.
