    1. Create `backend/scripts/debug_trace.py`:
       - File header comment (5-10 lines) explaining the trace CLI
       - Use `argparse` with subparsers for each trace command

    2. Implement argument parsing with these subcommands:
       - `request --id <trace_id>` — Single trace by ID
       - `errors --last <duration>` — Recent error traces (optional `--route <pattern>`, `--fingerprint <hash>`)
       - `task --id <celery-task-uuid>` — Celery task execution trace
       - `session --user <email> --last <duration>` — All traces for a user
       - `slow --threshold <ms> --last <duration>` — P95+ latency traces
       - `login --user <email>` — Login flow trace for specific user
       - All commands support `--production` flag and `--json` flag

    3. Implement `parse_duration(s: str) -> int`:
       - Parses "1h", "30m", "2d", "15s" into seconds
       - Pattern: `r'^(\d+)([smhd])$'` with multipliers: s=1, m=60, h=3600, d=86400

    4. Implement OpenObserve query functions:
       - `_get_base_url() -> str`: Returns `http://openobserve:5080` (local) or production URL based on `--production` flag
       - `_get_auth() -> tuple[str, str]`: Returns (email, password) from `OPENOBSERVE_ROOT_EMAIL` / `OPENOBSERVE_ROOT_PASSWORD` env vars. For `--production`, use `OPENOBSERVE_PROD_EMAIL` / `OPENOBSERVE_PROD_PASSWORD`.
       - `_search_traces(sql: str, start_time_us: int, end_time_us: int, base_url: str, auth: tuple) -> list[dict]`:
         - POST to `{base_url}/api/default/{stream_name}/_search`
         - Body: `{"query": {"sql": sql, "start_time": start_time_us, "end_time": end_time_us}}`
         - Stream name: Try `"default"` first. Note in comment that exact stream name for OTLP traces may need runtime discovery (see RESEARCH.md Open Question 2).
         - Use `httpx` with Basic auth
         - Return list of hit dicts from response

    5. Implement query builders for each subcommand:
       - `_query_by_trace_id(trace_id: str)`: `SELECT * FROM default WHERE trace_id LIKE '{trace_id}%' ORDER BY start_time ASC`
       - `_query_errors(duration_s: int, route: Optional[str], fingerprint: Optional[str])`: `SELECT * FROM default WHERE span_status = 'ERROR' ... ORDER BY start_time DESC LIMIT 50`
       - `_query_by_celery_task(task_id: str)`: `SELECT * FROM default WHERE celery.task_id = '{task_id}' ORDER BY start_time ASC`
       - `_query_user_session(user_email: str, duration_s: int)`: `SELECT * FROM default WHERE enduser.id LIKE '%{user_email}%' ... ORDER BY start_time DESC LIMIT 100`
       - `_query_slow(threshold_ms: int, duration_s: int)`: `SELECT * FROM default WHERE duration > {threshold_ms * 1000} ORDER BY duration DESC LIMIT 50`

    6. Implement `format_trace_timeline(spans: list[dict]) -> str`:
       - Group spans by trace_id
       - Build parent-child hierarchy using span_id and parent_span_id
       - For each trace, print header: `Trace {short_id}  Duration: {total_ms}ms  Status: {status}`
       - Print each span indented by depth:
         - Format: `  {timestamp}  [{service}]  {operation}  {duration}ms  {status}`
         - Indent 2 spaces per depth level
       - `short_trace_id(trace_id: str) -> str`: return first 12 characters

    7. Implement `format_json(spans: list[dict]) -> str`:
       - `json.dumps(spans, indent=2, default=str)`

    8. Implement `main(args: list[str]) -> None`:
       - Parse args, dispatch to appropriate query + formatter
       - Print to stdout

       - Test argument parsing for all subcommands
       - Test `parse_duration` with various inputs
       - Test `short_trace_id`
       - Test `format_trace_timeline` with mock span data (no OpenObserve needed)
