**Test structure:**
```typescript
describe("TEST-04: encryption performance benchmark", () => {
  let chatKey: Uint8Array;

  beforeAll(() => {
    // Random 32-byte AES-256 key
    chatKey = new Uint8Array(32);
    crypto.getRandomValues(chatKey);
  });

  it("encrypts and decrypts 100 messages within 2 seconds", async () => {
    // Generate 100 messages of realistic size (100-500 chars each)
    const messages = Array.from({ length: 100 }, (_, i) => {
      const padding = "x".repeat(100 + (i * 4)); // Variable length 100-496 chars
      return `Message ${i}: Test content for performance benchmark. ${padding}`;
    });

    // Warm-up: 5 operations to JIT-warm the crypto path
    const warmupKey = new Uint8Array(32);
    crypto.getRandomValues(warmupKey);
    for (let i = 0; i < 5; i++) {
      const ct = await encryptWithChatKey("warmup", warmupKey);
      await decryptWithChatKey(ct, warmupKey);
    }

    // Timed benchmark: encrypt all 100, then decrypt all 100
    const start = performance.now();

    const ciphertexts: string[] = [];
    for (const msg of messages) {
      ciphertexts.push(await encryptWithChatKey(msg, chatKey));
    }

    const plaintexts: string[] = [];
    for (const ct of ciphertexts) {
      plaintexts.push(await decryptWithChatKey(ct, chatKey));
    }

    const elapsed = performance.now() - start;

    // Verify correctness
    for (let i = 0; i < 100; i++) {
      expect(plaintexts[i]).toBe(messages[i]);
    }

    // Assert performance: under 2 seconds per D-03
    expect(elapsed).toBeLessThan(2000);

    // Log timing for CI visibility
    console.log(`[Performance] 100 encrypt + 100 decrypt: ${elapsed.toFixed(1)}ms`);
  });
});
```

Use `performance.now()` for timing (Research Pitfall 4 warns about CI variability, but the 2s threshold is generous for 200 AES-GCM operations).

```bash
#!/usr/bin/env bash
# check-file-sizes.sh — Report files exceeding 500-line threshold (TEST-05)
#
# Monitors encryption and sync source files for size regressions.
# Grandfathered files (in baseline) are listed but not flagged as new violations.
# Exit code: 0 always (report-only per D-08), but prints WARNING for new violations.
#
# Usage:
#   bash scripts/check-file-sizes.sh              # Full report
#   bash scripts/check-file-sizes.sh --ci         # CI mode: compact output, non-zero if new violations
#   bash scripts/check-file-sizes.sh --update     # Update baseline with current large files
```

**Configuration constants:**
```bash
THRESHOLD=500
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
BASELINE_FILE="$SCRIPT_DIR/.file-size-baseline.json"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
```

**Monitored directories (encryption/sync strictly, rest informatively per discretion):**
```bash
STRICT_DIRS=(
  "frontend/packages/ui/src/services/encryption"
  "frontend/packages/ui/src/services/chatSyncService"

)

  "backend/core/api/app/routes/handlers"
)
```

**Logic:**
1. Find all `.ts`, `.svelte`, `.py` files in monitored dirs
2. Count lines with `wc -l`
3. Filter files exceeding THRESHOLD (500 lines)
4. Read baseline JSON (if exists) to get grandfathered files
5. For each file over threshold:
   - If in baseline: print as "KNOWN: {file} ({lines} lines)" (not a new violation)
   - If NOT in baseline: print as "NEW VIOLATION: {file} ({lines} lines)" with a warning
6. Summary: total files checked, known large files, new violations

**Baseline JSON format (`scripts/.file-size-baseline.json`):**
```json
{
  "_comment": "Grandfathered large files as of 2026-03-26. Files listed here are known to exceed 500 lines and are not flagged as new violations.",
  "threshold": 500,
  "generated": "2026-03-26",
  "files": {}
}
```

The `files` object maps relative paths to line counts: `{ "frontend/packages/ui/src/services/chatSyncService/senders/index.ts": 123 }`.

**--update mode:** Scans all monitored dirs, writes current large files to baseline JSON.
**--ci mode:** If new violations found (not in baseline), exit with code 1. Print compact one-line-per-violation format.

**Generate initial baseline:** After creating the script, run `bash scripts/check-file-sizes.sh --update` to populate the baseline with current large files. This creates the grandfathered set.
