## Category A: Login OTP Failure

**Root cause:** `loginToTestAccount()` in `chat-test-helpers.ts` fails with "Login failed after 3 OTP attempts". The OTP generated by `generateTotp()` is rejected by the server 3 times in a row. This was confirmed by the api-keys-flow triage run (all 4 tests failed identically) and skill-web-search Phase 4.

Possible sub-causes:
1. TOTP clock drift between GHA runner and server (most likely -- GHA runners can have clock skew)
2. OTP secret rotation on the test account that invalidated the GHA secret
3. Server-side rate limiting on OTP attempts from GHA IP ranges
4. The TOTP window-boundary avoidance logic (lines 93-100 in chat-test-helpers.ts) is insufficient

**Fix strategy:** Investigate OTP timing. Add clock-drift compensation in `generateTotp()`. Consider increasing the TOTP validation window server-side for test accounts. Add retry with exponential backoff between OTP attempts.

**Evidence:** api-keys-flow (run 23649184159): all 4 tests show "Login failed after 3 OTP attempts". skill-web-search Phase 4 (run 23649180350): same error.

## Category C: Selector Drift

**Root cause:** CSS class selectors in test assertions reference DOM elements that no longer exist in the current frontend. This happens when components are refactored and class names change without updating test selectors.

**Evidence:**
- not-found-404-flow: `.not-found-options` not found (confirmed via GHA run 23649186932). All 4 tests in this spec fail because the Not404Screen component's recovery options container no longer uses this class.
- skill-news-search Phase 4: `.editor-content.prose` not found (confirmed via GHA run 23649181711). The chat message editor selector has changed.

**Fix strategy:** Inspect the current DOM for each affected component, find the correct selector, and update tests. Prefer data-testid attributes over CSS classes per testing conventions.

**Specs (5):**
- not-found-404-flow.spec.ts -- `.not-found-options` selector not found (confirmed via GHA)
- docs-links.spec.ts -- no login required, likely selector/routing drift on /docs/ pages

- seo-demo-chat.spec.ts -- no login required, likely selector/content drift on /demo/chat/ pages
- shared-chat-open.spec.ts -- no login required, likely selector drift on shared chat open flow

## Category D: Signup Flow Timeout

**Root cause:** The full signup flow test exceeds its 240-second timeout. The signup flow involves creating an account via Mailosaur email verification, which depends on external email delivery timing and potentially slow GHA runner performance.

**Evidence:** signup-flow (GHA run 23649182631): "Test timeout of 240000ms exceeded." Single test timedOut.

**Fix strategy:** Increase timeout for signup specs. Investigate which step is slow (email delivery? Mailosaur API? page load?). Consider splitting the flow into smaller focused tests. Check if Mailosaur polling interval is too conservative.

**Specs (4):**
- signup-flow.spec.ts -- "Test timeout of 240000ms exceeded" (confirmed via GHA)
- signup-flow-passkey.spec.ts -- uses Mailosaur + signup flow, likely same timeout issue
- signup-flow-polar.spec.ts -- uses Mailosaur + signup flow + Polar checkout, likely same timeout
- newsletter-flow.spec.ts -- uses Mailosaur for email verification, likely same timeout pattern

## Category E: Strict Mode Selector Violations

**Root cause:** Playwright's strict mode rejects `getByText()` calls that match multiple DOM elements. The status page renders the same text ("Groq", "Chat") in multiple locations (service name + component name, category name + test name), causing ambiguous matches.

**Evidence:** status-page (GHA run 23649185727):
- `getByText('Groq')` resolved to 2 elements (service name and component name)
- `getByText('Chat')` resolved to 5 elements (category name and individual test names)

**Fix strategy:** Use more specific selectors: `.service-name:has-text("Groq")` instead of `getByText('Groq')`, or use `getByRole()` with specific names. Update the mock data to avoid ambiguous text matches.

**Specs (1):**
- status-page.spec.ts -- strict mode violations for `getByText('Groq')` and `getByText('Chat')` (confirmed via GHA)
