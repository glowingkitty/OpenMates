       ```python
       rc = subprocess.run(cmd, capture_output=True, text=True, cwd=str(vdir))
       ```

       with:
       ```python
       try:
           rc = subprocess.run(cmd, capture_output=True, text=True, cwd=str(vdir), timeout=VITEST_TIMEOUT)
       except subprocess.TimeoutExpired:
           _log(f"  vitest timed out after {VITEST_TIMEOUT}s in {rel}", "WARN")
           overall_status = "failed"
           all_tests.append({
               "name": f"{rel}/vitest-timeout",
               "status": "failed",
               "duration_seconds": VITEST_TIMEOUT,
               "error": f"vitest timed out after {VITEST_TIMEOUT}s — likely a deadlock in crypto/jsdom tests",
           })
           continue
       ```

    The `continue` skips the JSON parsing for this timed-out run and moves to the next vitest directory (if any). The timeout entry gets recorded as a failed test so the pipeline knows vitest had an issue.

    IMPORTANT: Do NOT change any other part of run_tests.py. Only add the constant and modify the subprocess.run call with try/except.
    IMPORTANT: The `continue` must be inside the except block to skip lines 1065-1099 (JSON parsing of rc.stdout which does not exist after timeout).
