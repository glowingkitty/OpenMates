### Critical Gaps

#### 1. ARCH-03: Sender barrel not deployed (Phase 4)

`chatSyncServiceSenders.ts` remains a 2856-line monolith on dev with 8 direct `cryptoService` imports. Five sub-modules were created but are orphaned. The barrel re-export conversion (commit `2bcca807b`) appears to not have merged cleanly.

#### 2. OTEL-06: User opt-in toggle disconnected (Phase 6)

The Settings Privacy UI toggle saves `debug_logging_opted_in` to Directus, but no code reads this value and sets it as a span attribute. The privacy tier system (`user_tier.py`) checks for `enduser.debug_opted_in` on spans but it's never populated.

#### 3. OTEL-02/OTEL-05: Privacy tiers non-functional + incomplete instrumentation

TracePrivacyFilter always defaults to Tier 1 (most restrictive) since user attributes are never set on spans. Additionally, `key_received_handler.py` has no OTel spans.

#### 4. OTel trace output quality (user-reported)

`debug.py trace errors` output shows only root spans with no child span detail, HTTP paths, or meaningful context. The trace CLI needs rework to be practically useful.
